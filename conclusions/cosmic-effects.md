# Cosmic Effects — 技术方案

## 时光隧道（Warp Tunnel）

### 原理

星空粒子从屏幕中心向外辐射飞驰，模拟穿越虫洞的视觉。通过距离映射速度与亮度，制造 3D 透视深度感。

### Star 类核心字段

```
ox, oy       — 原始位置（动画阶段使用）
warpAngle    — 粒子相对于屏幕中心的角度（固定）
warpDist     — 粒子当前距中心的距离（每帧递增）
warpSpeed    — 当前飞行速度（随距离加速）
size         — 粒子大小
```

### 关键算法

**速度加速（透视感）**
```
每帧：warpSpeed += warpDist * 0.8 * dt    // 越远越快
      warpDist  += warpSpeed * dt
```

**亮度与大小映射**
```
t = warpDist / (屏幕对角线 * 0.5)          // 0~1 归一化距离
alpha = 0.15 + t * 0.85                   // 近暗远亮
size  = 0.8  + t * 3.5                    // 近小远大
```

**运动拖尾**
```
streak = t * 12
沿 warpAngle 方向画线段: (x+sx, y+sy) → (x-sx, y-sy)
近中心几乎无拖尾，远边界拉出长光痕
```

**循环重生**
```
if (warpDist > 屏幕对角线 * 0.8) {
    重新随机: warpAngle, warpDist(15~50px), warpSpeed
}
```
中心 15-50px 始终无粒子 — 形成隧道洞口。

**时空静止**
```
hoveredNebula ? dt * 0.08 : dt * 0.75
```
悬停星云时降速至 8%，隧道几乎停住；正常速度降低 25%。

## 闪电特效（Lightning）

### 原理

递归生长的树状结构，从屏幕边缘向中心弯曲延伸。严格控制每步角度变化，杜绝锐角折线。

### Lightning 类结构

```
sx, sy        — 根部（屏幕边缘随机点）
baseAngle     — 初始朝向（指向中心区域）
allSegs[]     — 所有线段 {x1,y1,x2,y2,trunk}
life/maxLife  — 生命周期 0.12~0.28s
```

### 关键算法

**递归生长（growTree）**

```
growTree(x, y, angle, length, depth):
    if depth == 0: return
    
    segLen = length * max(W,H) * 0.12~0.22
    steps = 3~6
    
    // 主干：每步角度微调 ±14°，平滑弯曲
    for i in 0..steps:
        angle += rand(-0.25, 0.25)    // 最大 ~14°
        (cx,cy) → (cx+cos(angle)*segLen, cy+sin(angle)*segLen)
        记录线段
    
    // 侧枝：从主干中间点分叉
    分叉角度: angle ± rand(0.7, 1.2)   // 40~70°，柔和
    递归调用: growTree(分叉点, 分叉角, length*0.3~0.6, depth-1)
```

**不出现锐角的保证**
- 每步角度变化：`rand(-0.25, 0.25)` 弧度 = ±14°
- 3-6 步累计弯曲也只在 ±50-84° 范围
- 远小于 100° 的约束

**树状层次**
- depth=4（主干）→ 1-3 个侧枝
- depth=3（粗枝）→ 1-3 个侧枝
- depth=2（细枝）→ 0-1 个侧枝
- depth=1（末梢）→ 停止

### 渲染三层

| 层 | 颜色 | 线宽 | 透明度 | 覆盖 |
|----|------|------|--------|------|
| 辉光 | `#6699dd` 蓝灰 | 6px | α×0.2 | 全部线段 |
| 中层 | `#aaddff` 亮蓝 | 主干3px/分枝1.5px | α×0.45 | 分层 |
| 芯线 | `#fff` 纯白 | 1.2px | α | 仅主干 |

`lineCap: 'round'` 让线段接头自然圆滑。

### 触发策略

```
间隔: rand(0.4, 4.5) 秒               // 高度随机
连击: 25% 概率一次劈 2~4 道              // 错开 40-150ms
时空静止: dt * 0.15 慢镜头展开           // 可观察分叉结构
清理: 超过 6 条时淘汰最旧的
```

## 性能考量

- 粒子数：200 颗星星 + 最多 6 条闪电 × ~50 线段 = ~500 绘制单元
- 纯 Canvas 2D，无 WebGL，移动端无压力
- 闪电只在 IDLE 阶段生成，动画阶段不消耗额外资源
- `bolts.filter(b => b.alive)` 每帧清理死链
