## 为什么测试工程师需要 Docker

- 测试环境一致性：本地 = CI = 生产
- 快速搭建依赖服务：数据库、消息队列、缓存一键启动
- 并行测试隔离：每个测试套件拥有独立环境
- 降低"在我机器上可以跑"的问题
