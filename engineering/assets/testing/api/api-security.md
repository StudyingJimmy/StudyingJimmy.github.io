# 常见 API 安全测试漏洞与防护策略

## OWASP API Security Top 10

> 调研进行中，以下为初步框架。

### 1. 对象级授权失效（BOLA）

攻击者通过修改请求中的资源 ID 访问不属于自己的数据。

**测试方法：**
- 替换请求中的用户 ID / 资源 ID
- 检查是否有权限校验返回 403

**防护：**
- 所有资源访问前校验所有权
- 使用随机 UUID 替代自增 ID

### 2. 用户认证失效

JWT 配置不当、弱密码策略、Token 泄露等。

**测试方法：**
- 检查 JWT 签名算法（禁止 none 算法）
- 测试 Token 过期策略
- 暴力破解测试

### 3. 过度数据暴露

API 返回了前端不需要的敏感字段。

**测试方法：**
- 对比 API 响应和前端实际使用的字段
- 检查是否泄露了密码哈希、内部 ID 等

### 4. 资源缺乏和速率限制

未限制请求频率导致可被 DDoS 或暴力枚举。

**测试方法：**
- 短时间内发送大量请求
- 验证是否有 429 状态码返回
